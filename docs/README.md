# Documentation Index

This folder organizes the repository as a compact systems engineering portfolio.

## Read In This Order

1. [portfolio_ideas.md](portfolio_ideas.md) for follow-on repository concepts.
2. `../reports/portfolio-roadmap.md` for the generated step-by-step multi-project roadmap and todo checklists.
3. [project_plan.md](project_plan.md) for the staged flagship build and publication roadmap.
4. [system_concept.md](system_concept.md) for the wildfire-drone mission context.
5. [verification_strategy.md](verification_strategy.md) for verification, validation, and review logic.

## Folder Purpose

- `data/` stores structured engineering records.
- `reports/` stores generated outputs from the audit tool.
- `src/` stores the reusable automation logic.
- `tests/` stores regression coverage for repository behavior.
- `data/portfolio_projects.json` stores the source of truth for all roadmap phases and todo items.
